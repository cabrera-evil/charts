CONNECTION DETAILS:

{{- if eq .Values.service.type "LoadBalancer" }}
1. Wait for LoadBalancer IP to be assigned:

   kubectl get svc {{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }} --watch

2. Once EXTERNAL-IP is assigned, connect using:

   Server IP: <EXTERNAL-IP>
   Port: {{ .Values.service.gamePort }}

3. Get the IP with:

   export SERVER_IP=$(kubectl get svc {{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }} -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
   echo "Connect to: $SERVER_IP:{{ .Values.service.gamePort }}"

{{- else if eq .Values.service.type "NodePort" }}
1. Get your node's external IP:

   export NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[?(@.type=="ExternalIP")].address}')
   export NODE_PORT=$(kubectl get svc {{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }} -o jsonpath='{.spec.ports[0].nodePort}')

   echo "Connect to: $NODE_IP:$NODE_PORT"

{{- else }}
1. This service is ClusterIP - not externally accessible.
   To expose it, upgrade with service.type=LoadBalancer or service.type=NodePort

{{- end }}

MANAGEMENT COMMANDS:

- Check server status:
  kubectl get deployment {{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }}

- View server logs:
  kubectl logs -f deployment/{{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }}

- Check persistent volumes:
  kubectl get pvc -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

- Restart server (force update):
  kubectl rollout restart deployment/{{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }}

- Access server console (if pod is running):
  kubectl exec -it deployment/{{ include "stardew-valley-server.fullname" . }} -n {{ .Release.Namespace }} -- /bin/bash

TROUBLESHOOTING:

If the server is not starting:
1. Check logs for Steam authentication errors
2. Verify Steam Guard code is current (expires after ~30 seconds)
3. Ensure your Steam account owns Stardew Valley
4. Check resource availability: kubectl describe pod -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "stardew-valley-server.name" . }}
